  "workflows"         → array WorkflowType[] (cards da app)

  "workflowCategories"→ string[]
  "subcategories"     → Record<string, string[]>

Operações: upsert por chave (SET), SELECT por chave (GET), DELETE.

API HTTP (Edge Function make-server-d8505aef):
  Base URL: https://rgbszdjyubbxhzhspuqv.supabase.co/functions/v1/make-server-d8505aef
  Auth: Header "Authorization: Bearer <anon_key>"

  GET  /data           → retorna todos os dados de uma vez
  POST /tools          → body: Tool[]         → sobrescreve array completo
  POST /prompts        → body: Prompt[]        → sobrescreve array completo
  POST /workflows      → body: WorkflowType[]  → sobrescreve array completo
  POST /categories     → body: {type, categories} → sobrescreve lista de categorias
  POST /subcategories  → body: {subcategories} → sobrescreve mapa de subcategorias

  brand_name    text NOT NULL
  source        jsonb  — {type: 'url'|'images'|'manual', ref?: string}
  palette       jsonb  — {primary, secondary, accent, neutral, background, text, extra[]}
  typography    jsonb  — {heading_font, body_font, weights[], scale}
  logo          jsonb  — {url, variants[], clear_space}

  outputs       jsonb — resposta do n8n
  created_at    timestamptz default now()

  Uso: upload de imagens associadas a prompts

8. WORKFLOWS N8N — CONFIGURAÇÃO (src/config/workflows.config.ts)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

N8N_BASE_URL: lido de localStorage('n8nBaseUrl') || VITE_N8N_WEBHOOK
              (padrão: http://localhost:5678/webhook)
              pode ser alterado em runtime via setN8nBaseUrl(url)

URL final de chamada: `${N8N_BASE_URL}/${workflow.webhookPath}`

Array WORKFLOWS (7 geradores):

1. extract-brand-style  (requiresBrand: false)
   Inputs: source_type (url|images), url, brand_name, images (até 6)
   Acção: analisa website/imagens e cria/actualiza brand_profile

2. generate-post  (requiresBrand: true)
   Inputs: theme, platform (IG/LK/FB/TK/X), goal, format, extra_context
   Output esperado: {caption, hashtags[], cta, visual_concept}

3. generate-visual  (requiresBrand: true)
   Inputs: concept, text_overlay, aspect_ratio
   Output esperado: {image_url, alt_text} ou {viewLink, downloadLink}

4. generate-carousel  (requiresBrand: true)
   Inputs: topic, num_slides (3/5/7/10), platform
   Output esperado: {slides: [{title, body, visual_concept}]}

5. repurpose-content  (requiresBrand: true)
   Inputs: source_url, source_text, platforms, num_posts
   Output esperado: {posts: [{platform, caption, hashtags[], visual_concept}]}
             ou: {variations: {platform: {caption, hashtags[]}}}

6. weekly-batch  (requiresBrand: true)
   Inputs: themes (textarea, um por linha), platform, post_frequency
   Output esperado: {calendar: [{day, theme, caption, visual_concept}]}

7. generate-bio  (requiresBrand: true)
   Inputs: platform (IG/LK/TK/X/YT), tone_extra
   Output esperado: {caption} (a bio gerada)

Funções auxiliares:
  getWorkflowById(id)        → WorkflowConfig | undefined
  getWorkflowsByCategory(cat)→ WorkflowConfig[]
  getWebhookUrl(workflow)    → string (URL completa)
  getCategories()            → ['N8N']

9. COMPONENTES DE SUPORTE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

WorkflowRunner (componente modal):
  - Recebe WorkflowConfig + BrandStyleProfile | null + onBack()
  - Renderiza formulário dinâmico a partir de workflow.inputs[]
  - Tipos de input suportados: text, textarea, select, image, email
  - Se hasImages: envia FormData (multipart); senão: JSON
  - Sempre inclui brand_id no payload se activeBrand existir
  - Se imagens: converte para base64 e adiciona campo ${field}_base64
  - Após sucesso: chama saveGeneration() → INSERT em generations
  - Renderiza ResultPanel com o output recebido

ResultPanel (componente de output):
  Campos do output reconhecidos:
    image_url / viewLink / thumbnailLink → ImageResult (com download e Drive)
    brand                                → BrandResult (paleta + JSON expansível)
    caption + hashtags[] + cta           → TextBlock por campo + "Copiar tudo"
    visual_concept                        → TextBlock
    variations: {plataforma: {caption, hashtags}} → PlatformVariations (tabs)
    slides: [{title, body, visual_concept}]       → SlideList
    posts: [{platform, caption, hashtags[]}]      → lista de posts por plataforma
    calendar: [{day, theme, caption, visual_concept}] → CalendarView
    message (fallback)                   → TextBlock genérico

BrandSelector:
  - Carrega brand_profiles via supabase.from('brand_profiles').select('*')
  - Ordena por updated_at DESC
  - Auto-seleciona a primeira marca ao carregar
  - Botão "+" → abre WorkflowRunner com workflow 'extract-brand-style'
  - Botão eliminar (hover) → supabase.from('brand_profiles').delete()

HistoryList:
  - Recebe brandId
  - Carrega generations por brand_id (limite 20, ordem DESC por created_at)
  - Lista colapsável: clique expande para ver inputs + output formatado
  - Data formatada com toLocaleString('pt-PT')

DesignAgentChat:
  - Estado: messages[], input, loading, sessionId (UUID fixo por sessão)
  - Envia para: ${N8N_BASE_URL}/design-agent
  - Payload: {message, brand_id, session_id, history (últimas 8 msgs)}
  - Timeout: 120 segundos (AbortSignal.timeout)
  - Resposta: {reply: string, actions?: [{workflowId, output}]}
  - Actions renderizadas inline como ResultPanel dentro da bolha do assistente
  - Enter = enviar; Shift+Enter = nova linha

KV layer interno:
  Usa SUPABASE_SERVICE_ROLE_KEY (não a anon key) para aceder à tabela
  kv_store_d8505aef com operações: upsert (SET), select (GET), delete (DEL)

Rotas:

  POST /workflows      → kv.set("workflows", body)
  POST /categories     → kv.set(key_por_type, categories)
  POST /subcategories  → kv.set("subcategories", subcategories)

14. FLUXO COMPLETO — DO CLIQUE AO RESULTADO
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

FERRAMENTA / PROMPT (CRUD):
  Utilizador preenche formulário modal
  → handleAddTool() / handleAddPrompt()
  → saveTools() / savePrompts() no AppContext
  → POST /tools ou /prompts na Edge Function
  → kv.set() na tabela kv_store_d8505aef

WORKFLOW (EXECUÇÃO):
  Utilizador seleciona marca (BrandSelector → brand_profiles)
  → clica num WorkflowConfig card
  → WorkflowRunner renderiza formulário
  → handleSubmit() monta payload (JSON ou FormData)
  → fetch(N8N_BASE_URL + webhookPath)
  → n8n processa e devolve JSON
  → setResult(data) → ResultPanel renderiza output
  → saveGeneration() → INSERT em generations

DESIGN AGENT (CHAT):
  Utilizador escreve mensagem
  → DesignAgentChat.send()
  → POST N8N_BASE_URL/design-agent com {message, brand_id, session_id, history}
  → n8n decide ferramentas com GPT-4o tool-calling
  → chama sub-webhooks internamente
  → devolve {reply, actions[]}
  → actions renderizados como ResultPanel inline

15. PONTOS IMPORTANTES / PARTICULARIDADES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

• O KV guarda arrays COMPLETOS — não há patch. Cada save substitui tudo.
• O n8n URL padrão é localhost:5678 mas pode ser alterado via
  localStorage('n8nBaseUrl') sem rebuild.
• Workflows "ativos" na app (lista KV) são filtrados por ACTIVE_WORKFLOW_PATHS;
  os outros são ignorados mesmo que estejam no KV.
• brand_profiles e generations usam o cliente Supabase (anon key) directamente
  — não passam pela Edge Function.
• Upload de imagens de prompts: bucket "prompt-images" (deve ser público).
• Cópia de prompt para clipboard: remove linhas que começam com "Model:" e
  "Casos de Uso:" antes de copiar.
• Categorias protegidas ('Todas', 'Todos') não podem ser eliminadas pelo utilizador.
• A subcategoria "Cinematografia" foi adicionada manualmente ao mapa de Design.
• Script de seed: seed-cinematografia.ts (executar com npx tsx seed-cinematografia.ts)
  — lê .env.local, faz merge idempotente por id, escreve em /prompts.
